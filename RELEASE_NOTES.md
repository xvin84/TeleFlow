# Release Notes

## [0.1.1-alpha] — 2026-03-24

### Исправлено

**Процесс не завершался после закрытия окна**
- Telethon-клиенты держали TCP-соединения открытыми после остановки Qt event loop, из-за чего `python` оставался в памяти
- Добавлен метод `TeleFlowApp.cleanup()`: последовательно останавливает планировщик, отключает все Telethon-клиенты (`client.disconnect()`), закрывает соединение с БД
- `__main__.py` вызывает `cleanup()` через `loop.run_until_complete()` внутри `with loop:` — то есть пока asyncio-цикл ещё доступен, но Qt уже завершён

**Windows: расписания не отображались и не удалялись**
- На Windows дефолтный `ProactorEventLoop` несовместим с `anyio` / `aiosqlite` в потоке планировщика — поток молча падал при инициализации, `add_schedule` возвращал пустую строку
- В методе `_thread_main` теперь явно устанавливается `WindowsSelectorEventLoopPolicy` перед созданием нового цикла
- Таймаут ожидания готовности планировщика увеличен с 5 до 10 секунд

**Расписания не отображались повторно в SendRulesDialog**
- `_render_schedules` / `_render_chats` вызывали `deleteLater()` без `removeWidget()` — виджеты оставались в layout как «призраки», индекс stretch-item сбивался, при повторном открытии диалога счётчик `lbl_sched_count` показывал 0
- Теперь перед `deleteLater()` явно вызывается `layout.removeWidget(row)`

**Race condition при выходе через кнопку «Выйти»**
- `_on_quit` в `dashboard.py` вызывал `ensure_future(scheduler.shutdown())` и сразу же `QApplication.quit()` — coroutine не успевала выполниться до остановки event loop
- Удалены вызовы `ensure_future` из `_on_quit` и `closeEvent`; вся очистка делегирована `app.cleanup()`

## [0.1.0-alpha] — 2026-03-17

Первый публичный релиз.

### Добавлено

**Авторизация и аккаунты**
- Авторизация через MTProto (Telethon): телефон → SMS-код → 2FA
- До 5 аккаунтов одновременно, быстрое переключение, статус online/offline
- Опциональный пароль приложения (bcrypt) с блокировкой UI при запуске
- Шифрование сессий: Fernet + PBKDF2 + уникальная соль на установку

**Шаблоны сообщений**
- CRUD шаблонов с форматированием (жирный, курсив, подчёркивание, зачёркнутый, моноширинный)
- Надёжное хранение code-форматирования через `QTextFormat.UserProperty`
- Медиавложения: до 10 файлов (изображения → album, прочее — поштучно)
- Импорт шаблонов из CSV

**Окно «Управление отправкой» (SendRulesDialog)**
- Единое окно для управления правилами отправки каждого шаблона
- Левая панель: назначение чатов с живым поиском и добавлением по клику
- Правая панель: расписания с поддержкой паузы, редактирования и удаления
- Кнопка «Отправить сейчас» внизу окна

**Планировщик**
- 5 режимов: один раз, ежедневно, по дням недели, с интервалом, случайное окно
- Персистентность задач (SQLite через APScheduler 4.x)
- Метаданные расписаний в `teleflow.db` (таблица `schedules`)
- FloodWait handler

**Интерфейс**
- Dark / Light тема, переключение без перезапуска
- Логи отправки (успех / ошибка / rate limit)
- Экспорт логов в CSV, фильтрация по статусу и поиск
- Системный трей (QSystemTrayIcon), уведомления при отправке
- Автозапуск (Linux / Windows)
- Локализация: ru / en

### Известные ограничения в 0.1.0
- Процесс не завершался при закрытии (исправлено в 0.1.1)
- Windows: расписания не работали (исправлено в 0.1.1)
