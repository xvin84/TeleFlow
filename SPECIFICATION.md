## 1. Обзор проекта

**TeleFlow** — десктопное приложение с графическим интерфейсом для автоматизации рассылки сообщений через аккаунты Telegram. Программа обеспечивает управление несколькими аккаунтами, гибкое расписание, поддержку медиавложений и импорт данных из CSV.

### Ключевые возможности

- Авторизация в Telegram через MTProto (Telethon) — поддержка обычных аккаунтов, **не ботов**
- Управление до 5 аккаунтов с быстрым переключением
- Выбор чатов (группы, каналы, личные, боты) с учётом прав пользователя
- Гибкий планировщик: одноразово, ежедневно, по дням недели, интервал, случайное окно
- Редактор сообщений с форматированием (Bold / Italic / Underline / Strike / Code)
- Медиавложения: до 10 файлов (изображения, видео, аудио, документы)
- Пароль приложения (bcrypt) с шифрованием сессий (Fernet + уникальная соль)
- Импорт чатов и сообщений из CSV
- Тема в стиле Telegram (тёмная / светлая, переключение без перезапуска)
- Системный трей, автозапуск, встроенные логи, системные уведомления *(в разработке)*
- Сборка в бинарники для Windows и Linux через GitHub Actions

## 7. Менеджер расписаний — детальное ТЗ

### 7.1 Проблема

Сейчас пользователь может **создать** расписание через `ScheduleWizard`, но не может:
- Видеть, какие расписания активны для шаблона
- Изменить время/режим существующего расписания
- Удалить расписание
- Приостановить без удаления

### 7.2 Предлагаемое решение — вкладка «Расписания» в редакторе

При выборе шаблона сообщения, под кнопками действий появляется секция **«Активные расписания»**:

```
┌─────────────────────────────────────────────────────────┐
│  📅 Активные расписания                    + Добавить   │
├─────────────────────────────────────────────────────────┤
│  🕐 Ежедневно в 10:00          [⏸ Пауза] [✏] [🗑]     │
│  🔁 Каждые 2ч                  [▶ Старт] [✏] [🗑]     │
│  📅 Один раз: 20.03.2026 15:30  (выполнено)   [🗑]     │
└─────────────────────────────────────────────────────────┘
```

### 7.3 Хранение расписаний

Добавить таблицу в `teleflow.db` для хранения метаданных о расписаниях:

```sql
CREATE TABLE schedules (
    id          TEXT PRIMARY KEY,           -- APScheduler schedule_id
    msg_id      INTEGER NOT NULL,
    account_phone TEXT NOT NULL,
    mode        TEXT NOT NULL,              -- one_time / daily_fixed / etc
    config_json TEXT NOT NULL,              -- сериализованный ScheduleConfig
    description TEXT NOT NULL,             -- human_description()
    created_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (msg_id) REFERENCES messages(id) ON DELETE CASCADE
)
```

### 7.4 Функциональные требования менеджера

| ID | Требование |
|---|---|
| SM-01 | При создании расписания — запись в таблицу `schedules` |
| SM-02 | Список расписаний для шаблона в редакторе сообщений |
| SM-03 | Кнопка удаления — удаляет из APScheduler + из `schedules` |
| SM-04 | Кнопка редактирования — открывает `ScheduleWizard` с предзаполненными значениями, при подтверждении заменяет расписание |
| SM-05 | Кнопка паузы/возобновления — вызывает `scheduler.pause_schedule` / `resume_schedule` |
| SM-06 | Статус: «активен», «на паузе», «выполнено» (для one_time) |
| SM-07 | При удалении шаблона — каскадное удаление всех расписаний |
| SM-08 | При удалении аккаунта — удаление всех его расписаний |

## 8. Редактор сообщений — детальное ТЗ

### 8.1 Текущее состояние

Редактор (`MessageEditorWidget`) работает, но имеет ограничения:
- Кнопка `{ }` (code) сохраняется через `fontFixedPitch`, что ненадёжно
- Нет многострочного превью отправляемого текста
- Нет секции управления расписаниями (см. §7)
- Нет счётчика символов

### 8.2 Улучшения редактора

#### 8.2.1 Панель инструментов

| Кнопка | Горячая клавиша | Telegram-тег | Статус |
|---|---|---|---|
| **B** — жирный | Ctrl+B | `<b>` | ✅ |
| *I* — курсив | Ctrl+I | `<i>` | ✅ |
| U — подчёркнутый | Ctrl+U | `<u>` | ✅ |
| ~~S~~ — зачёркнутый | Ctrl+Shift+X | `<s>` | ✅ |
| `{ }` — моноширинный | Ctrl+Shift+M | `<code>` | 🔶 починить |
| `❝` — цитата | Ctrl+Shift+Q | `<blockquote>` | ❌ добавить |
| `🔗` — ссылка | Ctrl+K | `<a href="">` | ❌ добавить |
| Счётчик символов | — | — | ❌ добавить |

#### 8.2.2 Исправление Code-форматирования

Проблема: `fontFixedPitch` теряется при Qt-сериализации. Решение: использовать `QTextCharFormat.setProperty(QTextFormat.UserProperty + 1, True)` и сохранять как `<code>` через custom property, либо отслеживать форматирование через CSS-класс в Qt HTML.

Простое надёжное решение — хранить форматирование как **plain Markdown** внутри, конвертировать при отображении и отправке:

```
хранение: **жирный** _курсив_ __подчёркнуто__ ~~зачёркнуто~~ `код`
отправка: to_telegram_html() → <b>жирный</b> <i>курсив</i> ...
отображение: markdown_to_rich_text() → QTextEdit через setHtml()
```

## 18. Глоссарий

| Термин | Определение |
|---|---|
| **MTProto** | Протокол Telegram для обычных аккаунтов (не ботов). Реализован в Telethon |
| **FloodWait** | Ошибка Telegram API при превышении лимита. Содержит время ожидания в секундах |
| **APScheduler** | Библиотека планировщика задач. v4.x использует anyio — запускается в отдельном потоке |
| **dispatch.py** | Реестр активных TeleflowClient + маршрутизация задач из scheduler-потока в main loop |
| **qasync** | Мост между PyQt6 event loop и asyncio. Использует `@asyncSlot` для async-слотов |
| **Fernet** | Симметричное шифрование (AES-128 CBC) из библиотеки `cryptography` |
| **app_salt** | Уникальный 32-байтовый случайный ключ, генерируется при первом запуске, хранится в БД |
| **ScheduleConfig** | Data-класс описывающий одно расписание. Умеет строить APScheduler Trigger |
| **RandomWindowTrigger** | Кастомный Trigger: каждый день в случайное время внутри окна HH:MM–HH:MM |
| **run_coroutine_threadsafe** | Стандартный asyncio API для запуска корутины из другого потока в заданный event loop |
